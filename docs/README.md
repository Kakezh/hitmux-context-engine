# Documentation

Hitmux Context Engine documentation is intentionally flat. Start with configuration, then use the other files only when you need client setup, troubleshooting, or package-level reference.

- [Configuration](configuration.md): conf config files, providers, Milvus/Zilliz, indexing, sync, and file filtering.
- [Quick Start](quick-start.md): Claude Code, Codex CLI, JSON clients, VS Code MCP, and local install examples.
- [Troubleshooting](troubleshooting.md): status checks, logs, reconnects, Windows `npx`, and common configuration problems.
- [Package Reference](package-reference.md): MCP tools and core TypeScript package usage.
